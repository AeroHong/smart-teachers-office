# 성취평가제 체크리스트 지원 도구 (ASA-support)

> 상태: 1차 구현 완료. 실 계정 로그인 상태에서의 브라우저 확인은 아직 못함(아래 "남은 확인" 참고).

## 실제 구현 위치

이 폴더(`tools/ASA-support/`)는 기획 문서 보관용이고, 실제 코드는 로그인·권한 제어가 필요해 포털에 통합했습니다.

- `apps/portal/src/pages/tools/asaUtils.js` — xlsx 파싱(추정분할점수/성적일람표) + 집계 계산 (순수 함수, Firestore 의존 없음)
- `apps/portal/src/pages/tools/AsaSupportCutoffs.jsx` — 관리자 전용, `/tools/asa-support/cutoffs`
- `apps/portal/src/pages/tools/AsaSupport.jsx` — 교사용 메인, `/tools/asa-support`
- Firestore: `/schools/{schoolId}/asaCutoffs/{grade_과목명}`, `/schools/{schoolId}/asaResults/{자동생성id}`
- `firestore.rules`: asaCutoffs(조회는 소속 교사 전체·쓰기는 관리자), asaResults(본인 업로드 건만, 관리자는 전체 조회)

## 배경

학기말 성적 마감 전, 나이스에서 성적 정보를 직접 확인하기 어려운 상황에서 성취평가제 체크리스트 항목(추정분할점수 대비 성취도 분포 등)을 미리 검토할 수 있도록 돕는 참고용 체크리스트 보조 도구.

## 업로드 파일 종류 (나이스 다운로드, 샘플: `tools/docs/`)

### 1) 추정분할점수 (관리자 업로드)

- 과목별 여러 행 블록 중 `학기말최종추정분할점수` 행만 사용 (A/B, B/C, C/D, D/E, E/미도달)
- 학년은 "N학년" 텍스트에서 추출 (단, "2026학년도"의 "6학년" 오탐 방지를 위해 `(\d)학년(?!도)` 패턴 사용)
- 3학년은 대상 아님 (2027년 추가 예정)

### 2) 성적 일람표 = 환산점수 데이터 (교과 담당 교사 업로드)

- 제목 셀에서 과목명(단위수) 추출 (예: `대수(4)`) — 추정분할점수의 과목명과 매칭 키
- 학생 데이터 행은 "반/번호" 셀(`\d+/\d+` 패턴, B열부터 시작하며 A열은 비어있음)을 전체 행에서 스캔해 위치를 찾음 — 컬럼을 고정 인덱스로 가정하지 않음
- 합계(총점)는 각 행에서 마지막으로 채워진 셀 값 사용 — 과목마다 수행평가 세부항목 개수가 달라 합계 열 위치가 달라지기 때문
- 자퇴생: 항목 중 "자퇴" 텍스트가 하나라도 있으면 제외 (합계 셀에 숫자가 남아있어도 제외)

## 고정분할점수 (추정분할점수 미제공 과목용, 관리자가 과목별로 수동 지정)

| 분류 | 경계값 |
|------|--------|
| 보통교과 공통과목 (공통국어·공통수학·공통영어·통합사회·통합과학·한국사) | A/B 90, B/C 80, C/D 70, D/E 60, E/미도달 40 |
| 과학탐구실험 | A/B 80, B/C 60, C/미도달 40 |
| 보통교과 선택과목 (2학년 과목 전체) | A/B 90, B/C 80, C/D 70, D/E 60 |
| 보통교과 체육·예술 교과(군) | A/B 80, B/C 60 |

## 분석 산출 항목 (교사 업로드 시 자동 계산, 미리보기 후 저장)

- 총 평가 대상 인원 (자퇴 제외)
- 성취도 A 대상 인원 / 비율
- 학기말 성적 교과평균
- 교과평균과 A-B 기준점수 비교 (대/소, 참고 정보용)
- 최저 경계값(D/E 또는 과목 카테고리의 최하위 경계) 미만 학생 목록 — **1학년 과목에만 표시/저장**

## 원본 데이터 미저장 원칙

- 성적 일람표(환산점수) 원본과 학생별 점수는 브라우저 메모리에서만 처리하고 Firestore에는 저장하지 않음. 저장되는 건 위 집계 항목뿐 (단, "최저 경계값 미만 학생" 명단은 요구사항에 따라 이름/반번호만 저장).

## 접근 제어

- 로그인 교사: 본인이 업로드한 결과(`asaResults`, `createdBy == uid`)만 조회/수정/삭제
- 분할점수 기준(`asaCutoffs`)은 관리자만 등록/수정, 교사는 조회만 (본인 과목 계산에 활용)
- 관리자는 전체 학교 결과 조회 가능

## 검증 상태

- `apps/portal/src/pages/tools/asaUtils.js`의 파싱/집계 로직을 `tools/docs/`의 실제 나이스 파일 4개(추정분할점수 1·2학년, 대수·물리학 성적 일람표)로 Node 스크립트 검증 완료 — 과목명/학년/학급/담당교사/학생별 총점/자퇴 제외/기준점수 매칭/집계 결과 모두 정상 확인.
- `npm run build` 컴파일 오류 없음.
- **남은 확인**: 브라우저 확장 미연결로 로그인 상태에서의 실제 화면 동작(업로드 UI, Firestore 저장/조회, 권한 분리)은 아직 사람이 직접 확인 못함. `npm run dev` 후 `/tools/asa-support`(교사) · `/tools/asa-support/cutoffs`(관리자)에서 실제 확인 필요.
