# 스마트 출결 시스템 기획서 (확장 버전)

## 추가 요구사항 (핵심 확장 기능)

### 1. 다목적 사용 구조
본 시스템은 단순 출결을 넘어 다양한 상황에서 활용 가능하도록 설계한다.

활용 예:
- 아침 조회 출결
- 교과 수업 출결
- 방과후 활동 출결
- 행사 참여 체크
- 특정 공간 입장 체크

👉 출결 시스템은 “고정 구조”가 아니라  
👉 이벤트 기반 구조로 설계

---

### 2. 학생 명단 입력 기능

#### 요구사항
- 교사가 직접 명단 입력 가능
- 스프레드시트 업로드 지원

#### 입력 방식
- Google Sheets 연동
- CSV 업로드
- 직접 입력

#### 필수 데이터
- 학번
- 이름
- 이메일

---

### 3. 출결 이벤트 설정 기능

출결은 이벤트 단위로 생성

#### 설정 항목
- 이벤트 이름
- 출결 유형
- 대상 그룹
- 시작 시간
- 종료 시간
- 장소
- 설명

---

### 4. 시간/장소 기반 제어

- 시간 제한 출결
- 장소 정보 설정
- 향후 위치 기반 확장 가능

---

### 5. 실시간 출결 현황 페이지

- 출석 인원 표시
- 미출석 인원 표시
- 실시간 갱신
- 교사용 수정 기능

---

### 6. DB 확장 구조

엔티티:
- Students
- Teachers
- Events
- AttendanceLogs

관계:
- Students ↔ Events
- Events → Logs

---

### 7. 핵심 구조

👉 이벤트 생성 → QR → 출석 → 실시간 확인

👉 Sheets 기반 → DB 확장 가능 구조
